\doxysubsection{Detailed Description}
This is a generic finite state machine used alongside classes derived from \doxylink{class_d_c_l_1_1_c_finite_state_base}{CFinite\+State\+Base}. 

Each state has three methods which need to be implemented (See \doxylink{class_d_c_l_1_1_c_finite_state_base}{CFinite\+State\+Base}) First create one or more classes derived from the base class and implement each of the three methods, on\+Enter(), on\+Exit() and on\+Active(). We create an object of this class and objects of each of our classes derived from the \doxylink{class_d_c_l_1_1_c_finite_state_base}{CFinite\+State\+Base} class. Then we add each of those objects to this one, specifying a unique name for each state. Initially, the state machine has no state and upon calling \doxylink{class_d_c_l_1_1_c_finite_state_machine_ad5b9c75d0f791a00704a0a6e19031b9b}{update()} will do nothing. We need to call switch\+To\+State(std\+::string str\+State\+Name) to set the initial state. Upon doing so, the state\textquotesingle{}s on\+Enter() method will be called, followed by it\textquotesingle{}s on\+Active() method, until the state machine is asked to switch to another state with a call of \doxylink{class_d_c_l_1_1_c_finite_state_machine_a09a913f7925bf6e3a847e4e331a4b1d2}{switch\+To\+State()} either externally, or from within a state itself. Usage example\+: First, create a couple of classes derived from the \doxylink{class_d_c_l_1_1_c_finite_state_base}{CFinite\+State\+Base} class which will be used for the FSM\textquotesingle{}s states
