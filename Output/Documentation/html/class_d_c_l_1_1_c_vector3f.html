◆ getDistanceSquared()

float DCL::CVector3f::getDistanceSquared ( const CVector3f & vec) const

Computes distance squared between this vector and the one given, treating each vector as a point in 3D space.

Parameters
vecThe other vector, as a position in 3D space
Returns
The computed squared distance between this and the given vector

This is faster than the getDistance() method, as no expensive square root is used. This is useful if you have many vectors/points and need to find the closest out of them all. After finding the closest, then you can squareroot the return value from this method or use getDistance() to obtain the true distance.

◆ operator+()

CVector3f DCL::CVector3f::operator+ ( const CVector3f & vec) const

Addition operator which adds two vectors together.

Parameters
vecThe other vector in the operation
Returns
The result

Vector addition is commutative. (v1 + v2 == v2 + v1) is always true. Triangle law of vector addition states that when two vectors are represented as two sides of the triangle with the order of magnitude and direction, then the third side of the triangle represents the magnitude and direction of the resultant vector. Example: CVector3f vOne(1.0f, 1.0f, 1.0f); CVector3f vTwo(2.0f, 2.0f, 2.0f); CVector3f vThree = vOne + vTwo;

Computes a new position projected from the origin position, along the given direction vector by the given float amount and stores the result in this vector.

Parameters
vOriginPositionThe position in 3D space where the origin is
vDirectionThe direction from the origin to project from the origin
fProjectionAmountThe amount along vDirection to move to the final computed position

vDirection doesn't have to be normalized, but it'd probably make more sense if it is. Use negative fProjectionAmount values to compute the projected position in the opposite direction Example...

CVector3f vOrigin(0.0f, 0.0f, 0.0f);
CVector3f vDirection(1.0f, 1.0f, 1.0f);
CVector3f vProjectedPosition;
vProjectedPosition.project(vOrigin, vDirection, 2.0f); // Would result in vProjectedPosition being 2.0f, 2.0f, 2.0f
